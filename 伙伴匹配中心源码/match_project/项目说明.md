# 伙伴匹配系统项目说明

更新时间：2026-02-09

## 1. 项目概览（小白版）
- 这是一个“找伙伴 + 组队”的项目  
- 后端是 Spring Boot + MyBatis-Plus，负责接口、业务规则、数据库操作  
- 前端是 Vue3 + Vite + Vant，负责页面展示和交互  
- 数据库用 MySQL，登录态用 Redis 存 Session  

## 2. 业务逻辑（用白话解释）
- 用户注册：填写账号、密码、星球编号 → 后端校验 → 把密码加盐后保存  
- 用户登录：账号密码校验通过 → 把“已登录”存到 Session → 前端拿到登录状态  
- 伙伴推荐：
  - 普通模式：分页拉用户列表  
  - 心动模式：用“标签相似度”算出最像的用户  
- 找伙伴：在前端选标签 → 调用标签搜索接口 → 返回符合标签的用户  
- 队伍系统：创建队伍、更新队伍、加入/退出队伍、解散队伍  
- 加密队伍：加入时必须填写队伍密码  
- 缓存预热：每天定时把推荐用户列表写入 Redis，提高速度  

## 3. 功能列表
### 用户模块
- 注册、登录、退出登录  
- 获取当前登录用户  
- 更新用户信息（含标签）  
- 管理员搜索用户  
- 按标签搜索用户  
- 推荐用户（分页）  
- 标签匹配用户（心动模式）

### 队伍模块

- 查询队伍（普通列表、分页）  
- 加入队伍（支持加密队伍密码）  
- 退出队伍、解散队伍  
- 我创建的队伍、我加入的队伍  

### 其它模块
- 缓存预热任务  
- 标签编辑与展示  

## 4. 业务流程（一步一步）
### 4.1 登录流程
1. 前端提交账号和密码  
2. 后端校验并用“盐值 + 密码”做 MD5  
3. 成功后把用户信息放到 Session  
4. 前端拿到成功响应，跳转到首页  

### 4.2 推荐流程
1. 前端调用 `/user/recommend`  
2. 后端先查 Redis 是否有缓存  
3. 有缓存就直接返回，没有就查数据库并写入缓存  

### 4.3 标签搜索流程
1. 前端选择标签  
2. 调用 `/user/search/tags`  
3. 后端在内存中过滤拥有这些标签的用户  

### 4.4 创建队伍流程
1. 前端提交队伍信息  
2. 后端校验人数、名称长度、过期时间  
3. 保存队伍表  
4. 同时写入“用户-队伍关系表”  

### 4.5 加入队伍流程
1. 前端点击加入队伍  
2. 后端校验队伍状态  
3. 如果是加密队伍必须提供密码  
4. 使用 Redisson 锁防止并发超员  
5. 写入用户-队伍关系表  

### 4.6 退出 / 解散流程
- 退出时如果队伍只剩 1 人 → 直接解散  
- 队长退出时 → 把队长转移给最早加入的用户

## 7. 接口文档（简明版）
基础路径：`/api`

### 7.1 用户接口 `/user`
- POST `/user/register`  
  - 入参：`userAccount` `userPassword` `checkPassword` `planetCode`  
  - 返回：新用户 id  
- POST `/user/login`  
  - 入参：`userAccount` `userPassword`  
  - 返回：用户脱敏信息  
- POST `/user/logout`  
  - 入参：无  
  - 返回：1  
- GET `/user/current`  
  - 入参：无  
  - 返回：当前登录用户  
- GET `/user/search`（管理员）  
  - 入参：`username`  
  - 返回：用户列表  
- GET `/user/search/tags`  
  - 入参：`tagNameList`（多值参数）  
  - 返回：匹配标签的用户列表  
- GET `/user/recommend`  
  - 入参：`pageSize` `pageNum`  
  - 返回：分页用户  
- POST `/user/update`  
  - 入参：用户字段（必须带 `id`）  
  - 返回：影响行数  
- POST `/user/delete`（管理员）  
  - 入参：`id`（请求体是数字）  
  - 返回：true/false  
- GET `/user/match`  
  - 入参：`num`（1～20）  
  - 返回：匹配用户列表  

### 7.2 队伍接口 `/team`
- POST `/team/add`  
  - 入参：`name` `description` `maxNum` `expireTime` `status` `password`  
  - 返回：队伍 id  
- POST `/team/update`  
  - 入参：`id` + 其他需要更新字段  
  - 返回：true/false  
- GET `/team/get`  
  - 入参：`id`  
  - 返回：队伍详情  
- GET `/team/list`  
  - 入参：`searchText` `status` `pageNum` `pageSize` 等  
  - 返回：队伍列表（含创建人信息、加入人数）  
- GET `/team/list/page`  
  - 入参：分页参数  
  - 返回：分页队伍  
- POST `/team/join`  
  - 入参：`teamId` `password`  
  - 返回：true/false  
- POST `/team/quit`  
  - 入参：`teamId`  
  - 返回：true/false  
- POST `/team/delete`  
  - 入参：`id`  
  - 返回：true/false  
- GET `/team/list/my/create`  
  - 入参：`searchText` 等  
  - 返回：我创建的队伍  
- GET `/team/list/my/join`  
  - 入参：`searchText` 等  
  - 返回：我加入的队伍
